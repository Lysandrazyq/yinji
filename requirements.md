# 开发需求

> 记录"印迹"做什么、做到什么程度、以后往哪走。改需求请更新本文件。

## 1. 目标与定位

一个轻巧、简洁、直观的**照片水印网页 App**，面向 iPhone（Safari）。给照片加上类似 OPPO / 徕卡的相机水印，第一版聚焦最简单的"白框黑字"样式。

## 2. 平台与形态

- **核心**：单文件网页 App（`watermark.html`），所有 UI 和逻辑都在里面。
- **PWA**：项目自带 `manifest.webmanifest` + 图标 + `sw.js`,**浏览器可以"安装到桌面 / 主屏"**,Win / 安卓 / iOS / macOS 全平台都能装,体验接近原生 App。
- **跨平台原生打包**（可选）：
  - Windows:用 Tauri 出 `.exe` 安装包(优先做,负责人主力机)
  - 安卓:用 Capacitor 出 `.apk` 安装包(优先做)
  - iOS / macOS:暂时只走 PWA(负责人没苹果电脑,不做原生打包)
- **核心原则**：核心 `watermark.html` 是唯一真相源。所有平台只是"套壳",改一处全平台同步。详见 `docs/cross-platform.md`。

## 3. 核心功能（第一版，已完成）

- **选择照片**：从相册选，或直接拍。
- **15 种预设水印样式 + 自定义模板，一键切换**：

  **基础白款**:
  - **经典**(classic):底部白栏 + 黑字,最克制简洁。
  - **拍立得**(polaroid):四周白边,底部更厚,复古文艺。
  - **宝丽来完全款**(polaroidPlus):加厚拍立得 + 米黄底 + 暖色调,做旧。
  - **极简框**(minimalFrame):细白边 + 下方小字,近乎隐形。
  - **杂志浅版**(magazineLight):白底 + 黑字大标题。

  **深色款**:
  - **黑底白字**(darkBar):底栏深黑 + 白字,夜景照片更搭。
  - **电影黑边**(cinema):上下黑色横条 + 白字,16:9 letterbox 感。
  - **杂志封面**(magazine):照片满版 + 底部暗渐变 + 白色大标题,时尚杂志风。
  - **哈苏风**(hasselblad):深色底栏 + 橙色细条 + 橙字。

  **彩色 / 特殊款**:
  - **徕卡**(leica):底栏顶部一道粗徕卡红横条 + 自动模式分隔线变红。
  - **节日红**(festivalRed):红色边框 + 金色文字,过年/节庆。
  - **复古胶片**(filmStrip):左右两侧加胶片齿孔,米黄底,胶卷感。
  - **签到打卡**(checkIn):底栏带红色定位图标,旅行打卡风。
  - **大字日期**(bigDate):照片上压巨大年份数字,大字报感。
  - **极简右下角**(minimalCorner):不加底栏,右下角一行小字。

- **自定义模板**:点搜索弹窗里的「+ 新建我的模板」 → 选布局(底栏 / 四周白边 / 满版渐变)→ 调底色 / 文字色 / 强调色 → 起名保存。保存在浏览器 localStorage,本机持久;在搜索面板里能找到、能编辑、能删除。
- **两种文字模式，一键切换**（和样式正交，可任意组合）：
  - 手动输入：用户自己写一句话（默认预填当天日期）。
  - 自动读取：从照片 EXIF 读出**拍摄时间、设备型号、光圈、快门、ISO、焦段**，排成相机水印样式。
- **实时预览**：改文字 / 切模式 / 切样式即时更新。
- **保存**：生成带水印图片，长按存入相册 / 下载。
- **模板搜索**：右上角搜索图标点开，弹出输入框 + 模板名单，按名字 / 描述 / 标签实时筛选（如「红」「复古」「杂志」「暖」），点一下应用并关闭。

## 4. UI 原则

简洁、直观为第一原则。主界面只暴露必要操作（选图 → 选模式 → 保存），不堆功能。具体视觉见 `design.md`。

## 5. 隐私

照片仅在本地处理，**不上传**到任何服务器。

## 6. 已知限制（第一版接受）

- 自动读取依赖照片自带 EXIF；截图、转发过的照片常丢失，读不到时自动切回手动输入并提示。
- 不含地点信息（GPS 转地名需额外服务，且常被系统抹除）。
- iPhone 的 HEIC 格式可能无法解析拍摄信息，遇到时走手动模式。

## 8. 决策记录（为什么这么定）

- **核心永远是网页**:负责人非程序员,网页零安装、零成本、改一处全平台同步。原生 App 只做"套壳",不是另起炉灶。
- **样式与文字模式正交**:样式控制外观,文字模式(手动 / 自动)控制内容,互不影响,组合更灵活。
- **自定义模板派生**:自定义模板不是从零写,而是基于"布局类型"(底栏 / 白边 / 渐变) + 颜色覆盖。这样实现简单,用户也好上手。
- **默认模式**:有 EXIF 时默认"自动读取"(更出彩),无 EXIF 时默认"手动输入"。
- **平台接口而非平台条件分支**:`window.Yinji.platform.*` 抽象保存 / 选图 / 存储等动作,各平台只是注入不同实现,核心代码不变。
