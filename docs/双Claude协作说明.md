## 三、你怎么和我对接

### 3.1 🔴 完成任务后 → 写交接条子（必做）

每次改完代码交付后，**你必须**写一个交接条子给我。方法：

**打开 `docs/工作交接.md`，按下面格式追加今天的内容：**

```markdown
## 工作交接 · YYYY-MM-DD

**完成项：**
- [x] 做了什么功能 / 改了什么东西
- [x] ...

**新版本号：** vX.X.X

**注意事项：**（有什么要让项目管家知道的）
- 比如：待部署、有 bug 待修、下次要做什么
```

写完后提醒南心：
```
今天的工作做完了！交接条子已写好。
去跟项目管家说「晚报」就行，他会自己读交接条子。
```

> ⚠️ 这个条子是**我和你之间的直接通道**——我晚报时会自动读取它，然后更新看板。不用靠南心传话。

### 3.2 开工前 → 建议南心看早报

如果南心一上来就说"帮我做XX"，而你注意到他还没看早报，可以提醒：

```
要不要先跟项目管家说个「早报」，看看今天优先级再开工？
```

### 3.3 不确定优先级时 → 让南心问早报

如果你不确定该先做哪件事，或者南心同时提了好几件事，建议：

```
这几件事的优先级我不确定，你可以问项目管家「早报」，让他帮你排一下。
```

## 四、你照常做的事（不变）

你该干嘛干嘛，这些规矩不变：

- 所有代码改动只改 `watermark.html`（单文件 + 零外部依赖）
- 改完后更新 `devlog/` 下的两份更新日志
- 更新 `CHANGELOG.md`（App Store 风格）
- 交付前跑 `python tools/md2docx.py` 重新生成 `.docx`
- 面向非程序员沟通（南心不是程序员）
- 照片绝不上传，隐私优先
- 改动前先确认需求
