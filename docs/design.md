# 设计规范

> 记录"印迹"长什么样、为什么这么定。调样式 / 配色 / 排版前请读本文件。

## 1. 设计方向

**"照片冲印工作室"** 的清爽、克制气质：暖白纸感背景 + 干净黑字 + 一抹赭橙点缀。与"白框黑字"成品调性一致。第一原则永远是**简洁、直观**——只暴露必要的操作。

## 2. 配色（CSS 变量）

| 变量 | 色值 | 用途 |
|---|---|---|
| `--bg` | `#EFEAE0` | 暖白纸感背景 |
| `--surface` | `#FFFFFF` | 卡片 / 面板 |
| `--ink` | `#1C1814` | 主文字（近黑） |
| `--ink-soft` | `#7A7065` | 次要文字 |
| `--line` | `#E2DACC` | 描边 / 分隔线 |
| `--accent` | `#C2632D` | 赭橙主强调色 |
| `--accent-soft` | `#F4E6D8` | 强调色浅底 |

圆角 `--radius: 22px`；卡片用柔和阴影；背景叠一层极淡的径向渐变增加层次。

## 3. 字体

- 界面正文 / 中文：系统字体栈 `-apple-system, BlinkMacSystemFont, "PingFang SC", …`（iPhone 上即苹方 + SF，原生精致）。
- 少量拉丁文点缀（如副标题）：衬线栈 `ui-serif, Georgia, "Songti SC", …`，增加编辑感。
- **不引入外链字体**（见技术说明的白屏教训）。

## 4. 界面结构

- 顶部：品牌名「印迹」+ 拉丁副标题（含版本号）；右上角并排两个按钮——「更多模板」药丸 + 设置齿轮图标（v2.9.0）。
- 空状态：一块虚线框的大点击区（选照片 / 拍照）。
- 编辑态：照片预览卡片 + 控制面板（模式切换 + 文字/信息 + 保存 / 换一张）。
- 结果：底部弹出 sheet，展示成品图 + 下载 / 关闭。

## 5. 水印排版规范（核心）

**通用规则**：文字黑色、系统字体；有效文字带（barH）≈ 画布宽度的 **12%**（最小 64px）。

- **手动模式**：单行文字，**居中**，字号 ≈ barH 的 0.34。
- **自动模式**（相机水印样式）：
  - 左块：设备型号（**加粗**）+ 其下方拍摄时间（灰色、小一号）。
  - 右块：参数串 `f/2.8 · 1/120s · ISO100 · 26mm`，**右对齐**，过长自动缩小字号。
  - 左右之间一条**竖直细分隔线**（仅在空间足够时绘制）。
  - 字段缺失时优雅省略；信息完全缺失时显示占位提示。

参数格式约定：光圈 `f/x`；快门 <1 秒写 `1/n s`；`ISOxxx`；焦段 `xx mm`；时间 `YYYY.MM.DD HH:MM`。

## 5.5 模板样式（35 预设 + N 个自定义，与文字模式正交）

每种样式都套用上面的"通用规则"画文字带（杂志款除外，因为它没有白栏），差别在外框结构。

### 经典（classic，默认）

照片下方直接一条白栏。白栏顶部有一条极淡的灰色细分隔线，避免与照片"贴死"。最克制，最适合作为默认。

### 拍立得（polaroid）

照片四周加白边：上 / 左 / 右各 ≈ 画布宽度的 **8%**，底部 ≈ **26%**（更厚）。文字带居中放在底部白边里，有效高度取底部白边的 55%。照片下加柔和阴影 + 一圈极淡描边，让"白照片配白边"也能看清照片轮廓。

### 徕卡（leica）

经典款基础上，在白栏**顶部**加一道徕卡红（`#E2001A`）横条，高度 ≈ barH 的 **14%**（醒目）；自动模式下左右文字之间的分隔线也改成徕卡红（粗一档），形成红色呼应。

> 早期试过在白栏左侧再加一道极细的纵向红条作为 Leica 红点的"无文字暗示"，但 canvas 在预览卡片里有 8px CSS 圆角，纵向红条延伸到画布底部会被圆角剪到（视觉上像断了/溢出），所以撤掉。横向红条本身已经足够辨识徕卡风格。

### 杂志封面（magazine）

照片**满版**（没有白栏）。顶部加一层短暗渐变 + 小号品牌签「印迹 · PHOTO」；底部加一层更深的暗渐变作为标题区，**白色加粗大标题**（手动模式 = 用户文字、自动模式 = 设备型号，统一转大写）+ 灰白色小副标题（自动模式 = 时间 · 参数串）。时尚杂志风，遮 / 不遮主体看用户照片构图。

### 宝丽来完全款（polaroidPlus）

加厚版拍立得：边距 ≈ **10% / 32%**；背景从纯白改成米黄 `#FAF6EE`；照片上叠 8% 透明度的赭橙暖色调（怀旧感）；照片加柔和投影；右下角衬线斜体写「印迹」字样作为落款。比标准拍立得更"做旧"、更出片。

### 黑底白字（darkBar）

经典款的反色版：底栏深黑 `#1A1A1A` + 白字 `#FFFFFF` + 灰白软字 `#A8A8A8`。夜景 / 深色照片更搭。

### 极简右下角（minimalCorner）

不加底栏。仅在照片右下角写一行小字(白色 + 模糊阴影增强可读性),最克制。

### 电影黑边（cinema）

照片上下各加一道黑色横条(高度各 ≈ 照片高的 10% / 14%),16:9 letterbox 感。文字白色置于下方黑条。

### 复古胶片（filmStrip）

左右两侧加宽度 ≈ 6% w 的胶片"身体"(深棕 `#1F1B16`),上面均匀打圆角白色"齿孔"模拟 35mm 胶卷。下方一道窄底栏,米黄字。

### 节日红（festivalRed）

照片外加红色边框(`#A40E0E`),边框宽 ≈ 2.8% w,底部更厚做文字区。文字金色 `#F5D078`,加粗,内圈再加一道金色细描边。过年/节庆专用。

### 签到打卡（checkIn）

底栏带渐变(米色 → 白),左侧画一个红色定位 pin 图标(纯 Canvas 路径,无图标字体依赖),右边接文字(左对齐,单行)。旅行打卡风。

### 大字日期（bigDate）

照片满版。底部叠暗渐变,左下角放一个**超大**的年份数字(字号 ≈ w*0.22,800 weight),下方再写完整日期(小一档)。大字报感。

### 哈苏风（hasselblad）

经典 + 暗色 + 橙色:底栏深 `#252320`,顶部一道极细橙色条 `#E8A050`(barH 7%),文字也是橙色。致敬 Hasselblad 联名相机款。

### 极简框（minimalFrame）

照片外加 ≈ 1.8% w 的细白边 + ≈ 8% w 的下边,照片画一道极细的灰色描边(透明度 22%)。下方右对齐写一行 inkSoft 灰字。"几乎没装饰"。

### 杂志浅版（magazineLight）

「杂志封面」的浅色变体:照片下方一整块白色面板(高 ≈ 18% w),小品牌签 + 黑色加粗大标题 + 灰色小副标题。同样的杂志气质,白底版更适合日常照片。

### 季节 / 节气款（springCherry / summerSea / autumnMaple / winterSnow，v2.7.1）

一套渲染器 `drawSeasonStyle(pal)` 配调色板 `SEASON`，每组 `{bg, text, soft, accent}`。结构：照片下方 tint 底色横栏（barH ≈ 13% w），照片边缘下方 `barH*0.24` 处画一道细 accent 横线（editorial 编辑感），线下放文字带。按季节 / 心情换色。初版四组：spring 粉 / summer 蓝 / autumn 橙 / winter 灰。**v2.9.1 又加三组同款变体**：毕业季（藏青字 `#2C3C68` + 金线 accent，学院风）/ 莫兰迪（低饱和灰玫 `#6E5D55`）/ 节气·青（青瓷绿 `#3F6B4C`）。新增样式不写新绘制代码,只加调色板再登记。**v2.10.0 再加四组**：中秋·月（暖金 `#F6ECD6` + 金线 `#C99A3C`，团圆 / 赏月，归「节日」分类）/ 雾蓝（莫兰迪雾蓝 `#3C5666`）/ 暮山（沉静暖褐 `#6E4B33`）/ 黛紫（藕紫黛色 `#534563`）。

### 相机品牌风（fujiFilm / sonyAlpha / nikonGold，v2.7.1）

一套渲染器 `drawBrandStyle(pal)` 配调色板 `BRAND`，结构镜像哈苏款：深色底栏 + 顶部一道品牌色 accent 细条（≈ barH 8%）+ 品牌色文字。初版三组：富士绿（墨绿底 `#16241D`）/ 索尼橙（黑底 `#141414`）/ 尼康黄（暖黑底 `#1A1712`）。**v2.9.1 加佳能红**：深黑底 `#1B1311` + 佳能红 accent 条 `#D8262C`，同样只加调色板再登记。**v2.10.0 加暗夜蓝**：深蓝底 `#0F1A28` + 亮蓝 accent 条 `#3E86D6` + 蓝字 `#6FB0EE`，夜景 / 科技感（用品牌渲染器但归「深色」分类，非真实品牌）。

### 玻璃底栏（glassBar，v2.7.1；磨砂 v2.8.0 重做）

液态玻璃：水印栏**叠在照片上**（不外扩画布）。栏区域下的照片做磨砂——**v2.8.0 改用 downscale-upscale 抹平法**（`frostRect`：把区域缩到极小的离屏 canvas 再开 `imageSmoothingQuality="high"` 平滑放大回来，降采样丢高频 = 等效强模糊）。原因：`ctx.filter="blur()"` 在 iOS Safari 的 canvas 2D 上几乎不生效，真机看着没模糊，故弃用。磨砂之上叠深色半透明渐变 `rgba(20,20,24,.26→.44)`（保证白字在任意照片上可读），顶部一道 `rgba(255,255,255,.42)` 高光线模拟玻璃边。本轮旗舰款，进主样式条。抹平法是 baseline 能力、全平台可用，无需降级守卫。

### 玻璃卡片（glassCard，v2.7.1）

液态玻璃变体：底部居中浮一块圆角磨砂 chip（宽 ≈ 74% w、高 ≈ 13% h、圆角 ≈ chip 高 34%）装文字。圆角路径用 `roundRectPath`（`arcTo` 画），磨砂（`frostRound`，同 v2.8.0 抹平法）裁剪到圆角内 + tint 渐变 `rgba(28,28,32,.30→.48)` + 白色高光描边。比 glassBar 更"卡片化"、更轻盈。

### 生日快乐（birthday，v2.7.1）

暖色渐变底栏 `#FFE7D2 → #FFF6F0`（barH ≈ 15% w）+ 顶部窄带 **10 个固定坐标的彩色纸屑圆点**（坐标写死,避免预览重绘闪烁）+ 棕字 `#7A4B2B`（weight 600）。生日 / 庆祝专用。

### 纪念日（anniversary，v2.7.1）

白底栏 + 居中一颗玫瑰色爱心 `#D2607A`（`drawHeartIcon` 两条贝塞尔曲线画）+ 玫瑰色文字 `#7C4A57`。恋爱 / 周年 / 温馨场景。

## 5.6 自定义模板

用户能自己捏一个模板:基于「布局类型」 + 颜色覆盖。

### 三种布局可选

- **底栏**(bar):照片下方一条横栏,可选顶部加一道强调色条(类似徕卡)。
- **四周白边**(frame):类似拍立得,但所有颜色和比例都用户控制。
- **满版渐变**(overlay):类似杂志,但渐变颜色和文字色由用户定。

### 可调参数

- **底色**(bg):底栏 / 边框 / 渐变末端颜色,色板从原生 `<input type=color>` 选。
- **文字色**(text):正文字色。
- **强调色**(accent,可选):底栏顶部 / 边框 / 渐变末端的细条颜色;开关默认关。

### 存储

写入 `localStorage` 的 `yinji.customTemplates.v1` 键。本机持久,清浏览器数据会丢。

### 在搜索面板里的呈现

- 自定义模板带一个赭橙 `我的` 徽章和右边的「删除」按钮。
- 选中后和预设一样用。
- 删除需二次确认。
- 当前用着的模板被删时,自动回退到「经典」。

> 加新预设样式时遵循同一接口:`drawXxx()` 负责画外框(自由发挥),常规底栏样式给出文字带矩形 `(ox, oy, w, barH)` 后调用统一的 `drawText`(内部按 `state.mode` 走手动 / 自动);杂志款 / 大字日期 / 极简右下角这类自定义布局则直接自己画文字。`DRAW_DISPATCH` 字典加一条 `id -> 函数` 映射,`TEMPLATES` 数组加一条登记记录,就完事。技术细节见 `tech.md`。

## 5.6 「更多模板」面板

右上角按钮显示**文字「更多模板」**（图标 + 文字，不再是纯图标），点开后弹出底部 sheet。标题「更多模板」，副标题「按分类挑，或输入关键词搜」。面板自上而下：

- **分类标签行（`.cat-row`）**：8 个胶囊标签——全部 / 基础 / 深色 / 彩色 / 季节 / 品牌 / 玻璃 / 节日。点一下只看该类；选中态用赭橙 `--accent` 填充。模板多了之后，先按分类收窄再搜，比纯搜索更快。在 375 宽屏上自动换行成 2 排。
- **输入框**：输入关键词实时过滤（匹配名称、描述、标签、id）。和分类标签叠加生效（先按类、再按词）。
- **名单**：每项一张卡片，含「名称 + 描述」；当前选中的会高亮 + 标"当前"小徽章。**每张卡右侧有「☆ 常用 / ★ 常用」星标**（v2.8.1），控制它是否出现在主界面顶部快捷栏。
- 点一下卡片：立刻应用该样式 + 关闭弹窗 + 顶部 toast 确认。点星标只切换"是否常用"，不切换样式（事件 `stopPropagation`，不冒泡到选样式）。
- 关键词支持中英文标签：如 `红 / leica`、`复古 / polaroid`、`杂志 / magazine`、`暖`。
- 点遮罩或「关闭」退出；每次打开都重置回「全部」分类。

> 分类与模板 id 的对应写在 `CAT_IDS`（每类一组 id），由它反推出 `CAT_OF`（id→类）。8 个分类刚好覆盖全部 26 款内置模板（基础 6 / 深色 4 / 彩色 4 / 季节 4 / 品牌 3 / 玻璃 2 / 节日 3）；用户自定义模板只在「全部」里出现，不归入内置分类。

## 5.7 编辑区底色

`.canvas-stage` 背景从纯白改成米白 `var(--bg)`，并加一圈淡色描边。这样不管是拍立得 / 宝丽来的白边，还是经典 / 徕卡的白栏，都能和卡片背景区分开，预览不再"白吃白"。Canvas 本身加一层轻投影帮助视觉对比。

## 5.8 设置面板（v2.9.0）

右上角齿轮图标打开底部 sheet，标题「设置」、副标题「下面这些选择都只存在你这台设备上」。面板可滚动（`max-height:86vh; overflow-y:auto`），按功能分六段（每段一个赭橙小标题 `.set-sec > h4`）：

- **外观**：主题三选一 seg（跟随系统 / 浅色 / 深色）+ 液态玻璃开关（从「更多模板」面板搬来）。
- **打开照片时**：记住上次选择开关；默认文字模式 seg（用上次的 / 手动 / 自动）；默认水印文字 seg（今天日期 / 留空 / 固定一句）+ 选「固定」时下方展开一个输入框；**默认打开模板下拉（v2.9.1）**（`.set-select`，「用上次用的」+ 全部预设 + 自定义模板）。
- **自动模式显示哪些信息**：6 个复选框组成网格（`.set-grid` / `.set-check`），设备 / 时间 / 光圈 / 快门 / ISO / 焦段，各自可勾掉。
- **照片与导出**：默认导出画质 seg；导出尺寸上限 seg（省流 / 标准 / 高清）；长按看原图开关。
- **数据与隐私**：竖排三个按钮——重置主样式条常用栏、清空我的自定义模板（偏红 `.btn-warn`）、恢复默认设置（偏红）；下方一段隐私说明（强调本地、不上传）。
- **AI · 即将推出（v2.9.1，占位预告）**：一段说明（3.0 将加入智能文案 / 自动排版等，并重申「照片不上传」底线、联网能力需单独征得同意）+ 一颗「AI 功能抢先看 ✨」ghost 按钮，点了只弹 toast 预告，本版不产生任何 AI 行为。为 3.0 主线做视觉与心理铺垫。
- **关于**：版本号 + 「重看新手引导」按钮。

控件全部复用既有的 `.seg` / `.switch` / `.btn` 样式；破坏性动作用 `.btn-warn`（文字偏红，深色模式另调更亮的红）并二次确认。底部「完成」主按钮关闭。

## 5.9 新手引导（v2.9.0）

首次打开自动弹出的底部 sheet，分步卡片式（不是一次性长文）。每张卡片：居中大 emoji + 标题 + 一段说明，下方一行进度圆点（当前点用赭橙放大）。底部「跳过」（文字按钮）+「下一步」主按钮；最后一步按钮变「开始用」、隐藏「跳过」。共 4 步：① 欢迎 + 强调本地处理不上传 ② 选图 + 常用栏 / 更多模板挑样式 ③ 手动 / 自动两种文字 ④ 保存到相册 / 复制图片 + 齿轮设置入口。看过一次后本机记住（`yinji.onboarded.v1`）不再弹；设置「关于」可重看。**故意不**点背景关闭，确保首跑用户做出选择。

## 6. 交互细节

- 按钮按下有轻微缩放反馈。
- 所有状态（载入中 / 成功 / 失败原因）都给用户可见提示，不静默。
- 主操作"保存图片"用赭橙主按钮；次操作"换一张"用浅底次按钮。
- **长按预览看原图（v2.8.0）**：编辑页按住预览图 ≥ 220ms 临时显示无水印原图，松手恢复；移动 > 10px 视为滚动、取消触发。Canvas 加 `-webkit-touch-callout:none; user-select:none; cursor:zoom-in;` 压掉 iOS 长按系统菜单。图下方一行小字 `.stage-hint`「长按图片可看原图对比」做引导。

## 7. 状态栏配色（v2.8.0）

手机状态栏颜色由 `<meta name="theme-color">` 决定。页面顶部因 `radial-gradient(… rgba(255,255,255,.65) …)` 高光 + `background-attachment:fixed`（顶端恒接近纯白），原奶油色 `#EFEAE0` 与之"两张皮"。**v2.8.0 调亮到 `#F4F0E8`**（奶油底色与顶端近白高光的折中），状态栏与页面顶端基本融为一体。

> v2.9.0 起 `theme-color` 不再写死：JS 根据深 / 浅模式动态切换——浅色仍用 `#F4F0E8`，深色用 `#16130F`（见 §8）。

## 8. 深色模式（v2.9.0）

设置「外观」里三选一：**跟随系统 / 浅色 / 深色**（默认跟随系统）。

**实现思路——只换界面外壳，绝不动水印成图。** 深色仅给 App 的界面（卡片 / 弹窗 / 按钮 / 文字）换一套暖色深色调色板，**Canvas 画出来的水印照片本身完全不受影响**：用户做的图是什么白框黑字、什么底色，深浅模式下导出完全一致。这是刻意的——水印是要发出去 / 存相册的成品，不能因为用户当时开了深色就变样。

**配色落地——CSS 变量 + `body[data-theme]` 覆盖。** 调色板写在 `:root` 的 CSS 变量里（`--bg / --surface / --ink / --ink-soft / --line / --accent / --glass-bg …`）；深色用 `body[data-theme="dark"]{ … }` 整体改写这些变量为暖色深调（背景 `#16130F`、卡片 `#221E19`、文字 `#F1EBE1`、赭橙强调 `#E08A4C`），并对少数写死浅色的地方（输入框 / 分类标签 / 下拉 / 提示 / canvas 舞台底 / toast 等）补了深色覆盖块——和既有的 `body[data-glass="off"]` 覆盖是同一套写法。深 + 关玻璃用更高优先级的组合选择器 `body[data-theme="dark"][data-glass="off"]` 兜底，不依赖样式表书写顺序。

**「跟随系统」的判定在 JS 里做。** `applyTheme()` 用 `window.matchMedia('(prefers-color-scheme: dark)')` 把「跟随系统」解析成明确的深 / 浅，再给 `<body>` 写死 `data-theme="dark"` 或 `"light"`——这样 CSS 只需处理两种确定状态。同时同步切换 `<meta name="theme-color">`（深 `#16130F` / 浅 `#F4F0E8`），让手机状态栏跟着变。系统深浅切换时有监听器实时重绘，但**仅当用户选的是「跟随系统」**才跟着动；用户手动锁了深或浅就不受系统影响（老 Safari 用 `addListener` 兜底）。
