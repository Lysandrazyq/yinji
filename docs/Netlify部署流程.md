# 把「印迹」部署到 Netlify · 一步步图文流程

> 给南心看的傻瓜流程。Netlify 和你现在用的 GitHub Pages 一样，都是**免费托管静态网页**的平台。
> 你的「印迹」是纯静态网页，搬过去**代码一个字都不用改**，只是换个地方放文件。

---

## 0. 先想清楚：到底要不要换？

- ✅ **值得换**，如果你图：**上传更省事**（拖文件夹 vs 一个个传）、**网址更短好记**（`yinji.netlify.app`）、**传错能一键回滚**。
- ❌ **解决不了**：如果你图"国内打开更快/更稳"——Netlify 的 `netlify.app` 在国内**同样可能慢甚至打不开**，不保证比 GitHub 强（这是"墙"的问题，换平台治不了）。真要国内稳，得用国内云 + 域名备案，对个人太折腾，不建议。

> 一句话：**换 Netlify 是为了"你自己上传/维护更轻松"，不是为了"用户打开更快"。**

---

## 1. 要传哪些文件（重要）

把 `yinji` 文件夹里**这些**传上去就行（其它是开发用的，不用传）：

**要传 ✅**
- `index.html`（新加的，让根网址能直接打开）
- `watermark.html`（App 本体）
- `sw.js`（离线缓存）
- `manifest.webmanifest`（"添加到主屏幕"用）
- `icons/` 整个文件夹（图标）
- `CHANGELOG.md`（更新日志，可选）

**不用传 ❌**：`docs/`、`devlog/`、`tools/`、`CLAUDE.md`、`package.json`、`_check.js`（这些是开发/文档，传了也不影响，但没必要）

> 最省事的做法：把上面"要传"的几样单独拷到一个干净文件夹（比如桌面建个 `yinji-上线` 文件夹），整个拖上去。

---

## 2. 方式 A：拖拽部署（推荐，最简单，不用懂 Git）

1. 电脑浏览器打开 **netlify.com** → 右上角 **Sign up** 注册（用邮箱、或直接用 GitHub / Google 账号登录都行，免费）。
2. 登录后，找到 **"Add new site"（新建站点）→ "Deploy manually"（手动部署）**；或直接打开 **app.netlify.com/drop**。
3. 把你第 1 步准备好的那个文件夹，**整个拖进网页上那个虚线框里**，松手。
4. 等几秒——它会给你一个随机网址，像 `https://shiny-cat-123.netlify.app`。点开就是你的「印迹」了 🎉
5. **改成好记的网址**：进这个站点 → **Site configuration（站点设置）→ Domain management / Site details → Change site name（改站点名）**，改成比如 `yinji`，网址就变成 **`https://yinji.netlify.app`**（名字被占用就换一个）。

✅ 完成！这个网址就能发给别人用，也能在 iPhone Safari 打开后"添加到主屏幕"当 App。

---

## 3. 以后怎么更新（我每给你一版新的，你这样做）

1. 回到 Netlify，进你这个站点的 **"Deploys"（部署记录）** 页。
2. 页面下方一般有一句 *"Drag and drop your site folder here"*（把站点文件夹拖到这里更新）。
3. 把**新的那个文件夹**整个拖进去 → 等几秒 → 自动变成最新版。

> 跟 GitHub 比，省了"一个个文件覆盖"的步骤，整文件夹一拖就好。

---

## 4. 传错了想退回（一键回滚）

1. 进 **"Deploys"** 页，能看到每次部署的历史记录。
2. 点你想退回的那一次（比如上一版）。
3. 点 **"Publish deploy"（发布此版本）** → 网址立刻变回那一版。

> 这是 Netlify 比 GitHub Pages 方便的地方：手滑传错也不怕。

---

## 5. 方式 B：连 GitHub 自动部署（进阶，可跳过）

如果你以后想"我一更新 GitHub，网站自动更新"：注册时选 **"Import an existing project" → 连接 GitHub → 选你的 yinji 仓库**，以后往 GitHub 一推，Netlify 自动发布。**但这要懂点 Git，你现在用拖拽就够了，这条先不用管。**

---

## 6. 换地方后要记得的 3 件事 ⚠️

1. **新网址要重新发给大家**：老用户在 `github.io` 那个网址"添加到主屏幕"的图标，**还指向老地方、不会自动变**。把新网址 `yinji.netlify.app` 重新发一遍，让大家重新"添加到主屏幕"。（你也可以两个网址都先留着，老的不用关。）
2. **百度统计后台加新域名**：去 `tongji.baidu.com` → 站点管理 → 把 `yinji.netlify.app` 也加进去，数据照常记。（站点 ID 不用换，代码里那串照旧。）
3. **隐私 / 红线不变**：还是纯静态、照片只在本机处理、不上传——换托管平台不影响这些。

---

## 7. 关于根网址（技术备注，你不用管）

你的 App 本体是 `watermark.html`，不是 `index.html`。我已经在文件夹里**加了一个小小的 `index.html`**：有人打开根网址（`yinji.netlify.app`）时，它会自动跳到 `watermark.html`。所以根网址、和带 `/watermark.html` 的网址，**两个都能用**。（这个小文件对你现在的 GitHub Pages 也有好处：根网址 `…/yinji/` 也能直接打开了，离线缓存也更稳。）
